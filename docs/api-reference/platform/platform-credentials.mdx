---
title: "Platform Credentials"
description: "Secure, per-platform credentials (API keys, secrets, tokens)"
---

Credentials store the secrets needed to talk to external marketplaces. Values are redacted in responses to avoid accidental leakage.

## Data shapes

```json
// PlatformCredentialCreateRequest
{
  "platformId": "c2a393da-5ef5-4c35-9b65-22a932224b7f",
  "credentialKey": "apiKey",
  "credentialValue": "encrypted-or-kms-ref",
  "isActive": true
}

// PlatformCredentialResponse
{
  "id": "6b7a7dc0-f1a6-4660-b50b-4ab1781d3a5e",
  "platformId": "c2a393da-5ef5-4c35-9b65-22a932224b7f",
  "platformName": "Amazon",
  "credentialKey": "apiKey",
  "credentialValue": "***REDACTED***",
  "isActive": true
}
```

## Endpoints

| Method & Path | Description |
|---------------|-------------|
| `GET /api/platform-credentials` | Optional filters: `platformId`, `isActive`. Returns a list. |
| `GET /api/platform-credentials/{id}` | Fetch a single credential. |
| `POST /api/platform-credentials` | Create (returns `201`). |
| `PUT /api/platform-credentials/{id}` | Update `credentialKey`, `credentialValue`, and/or `isActive`. |
| `DELETE /api/platform-credentials/{id}` | Remove the credential (`204`). |

### Specialized queries

- `GET /api/platform-credentials?platformName={name}` optionally combined with `isActive=true|false`
- `GET /api/platform-credentials/by-key?platformId={uuid}&credentialKey={key}` to fetch a unique key/value pair for a platform

Responses always redact the actual secret; store the value returned from `POST` or `PUT` if clients need to echo it elsewhere.
